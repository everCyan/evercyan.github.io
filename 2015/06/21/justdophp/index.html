justDoPHP 快速开发

By 严宇川

2015-06-21 更新日期:2015-06-21

justDoPHP 简介

gitHub : https://github.com/everCyan/justDoPHP

  • 这不是一个PHP框架
  • 这也不会减少你的代码工作量
  • 这也许反而会增加你的开发时间
  • 以上

先说下我为什么折腾这个东西.
平常开发中, 经常会需要写一些小的工具或者小的网站, 但开发方案无非两种.

  • 用成熟的PHP框架开发
  • 原生的PHP写法

我讨厌用第一种, 非常折腾, 且感觉毫无必要, 一个简单的页面, 也要搞个MVC, 要熟悉框架规则下的模板引擎解析规则…

但我也讨厌用第二种, 一堆的 PHP 文件暴露在项目目录下, 无组织无纪律的各种 include, 需要引用的工具包各种乱放, 令人鼻酸. 访问的 URL 是暴露出来的各种 php 结尾的文件地址.

没有逼格!

没有逼格!!

没有逼格!!!

于是, 折腾了下这个东西出来了.

简单说下这个项目

  • 项目所需要用到的工具, 像路由, DB, LOG, 缓存, 页面错误, 以及种种, 全部依赖于 PHP 的 Composer 包管理.
  • 项目前端全部使用 Bootstrap 展现, 程序员最讨厌写CSS, 你懂的.
  • 项目开发中的代码, 全部是原生的 PHP + HTML 各种混排, 只要写的爽, 不管MVC.

当然, 这只是个人的一种解决方案.

恩, 就是这样.

justDoPHP 目录结构

  • config
    • bootstrap.php
      //启动文件, 包括所有包的初始化
    • config.php
      //全局配置文件, 包括所有包的配置文件
    • function.php
      //全局公共函数文件
    • routes.php
      //全局路由文件, 所有URL解析在此执行
  • public
    • bootstrap
      //引入 bootstrap 前端框架
    • layout
      //布局文件
    • log
      //日志目录
    • page
      //所有页面存放目录, 通过路由指向引入
    • style
      //自定义样式, 图片, JS等
  • vendor
    //composer 包目录
  • .htaccess
  • composer.json
    //composer 包管理文件
  • index.php
    //入口文件
  • readme.md
    //关于文档

关于包的使用

鉴于个人目前使用习惯, 只集成了以下几个包.

  • 路由
    composer, “noahbuscher/macaw” : “dev-master”
    gitHub, https://github.com/noahbuscher/Macaw
    demo, 参照代码中的 routes.php 文件

  • DB
    composer, “illuminate/database”: “*”
    gitHub, https://github.com/illuminate/database
    demo, 参照该包的文档:

    ).select
    $results = DB::select('select * from users where id = ?', [1]);
    $results = DB::select('select * from users where id = :id', ['id' => 1]);
).insert
    DB::insert('insert into users (id, name) values (?, ?)', [1, 'Dayle']);
).update
    DB::update('update users set votes = 100 where name = ?', ['John']);
).delete
    DB::delete('delete from users');
).一般语法
    DB::statement('drop table users');
).监听查找事件
    DB::listen(function($sql, $bindings, $time) {
    });

).数据库事务处理
    DB::transaction(function() {
        DB::table('users')->update(['votes' => 1]);
        DB::table('posts')->delete();
    });
    DB::beginTransaction();
    DB::rollback();
    DB::commit();

).使用多个连接
    $users = DB::connection('foo')->select(...);
).取用原始底层的 PDO 实例
    $pdo = DB::connection()->getPdo();
).重连接
    DB::reconnect('foo');
).关闭连接
    DB::disconnect('foo');

  • 错误处理
    composer, “filp/whoops” : “*”
    gitHub, https://github.com/filp/whoops
    demo, 代码文件中任何地方执行 : 

$msgStr = "404 Not Found";
error($msgStr);

  • 日志
    composer, “monolog/monolog”: “1.2.*”
    gitHub, https://github.com/Seldaek/monolog
    demo, 在 function.php 中对日志作了简化

    Log::info("Log Info Begin");
Log::error("Log Info Begin");
Log::debug("Log Info Begin");

其余的包可通过 https://packagist.org/ 查找安装, 扩展 so easy, 老板再也不用担心我重复造轮子了.
